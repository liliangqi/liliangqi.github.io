Ubuntu 16.04安装caffe及torch

2017-05-03 machine learning

　　为了满足计算资源的分配，近期实验室新进了一批配备高端显卡TITAN X (PASCAL)的计算机，原始搭载系统为Ubuntu 14.04，在我的钻(zhe)研(teng)下不小心搞崩了系统，只能重装。在选择系统版本时考虑到刚出的17.04可能不够稳定，许多软件包的支持也不完善，而14.04又不够炫酷，索性装个16.04好了。

　　在人工智能和计算机视觉大红大紫的今天，越来越多的同学纷纷踏入深度学习的坑。俗话说工欲善其事，必先利其器，所以今天给大家介绍一下当下十分流行的深度学习框架caffe和torch在Ubuntu系统下的安装方法，也算是对我近期折腾新机器的一点总结。

安装Ubuntu

  1. 在Ubuntu官网下载系统镜像文件（.ios）
  2. 在windows系统下下载ultraISO，用于制作U盘启动盘
    1. 打开ultraISO，文件->打开，选择刚才下载的iso文件
    2. 启动->写入硬盘映像，选择自己的U盘
    3. 便捷启动->写入新的驱动器引导扇区->Syslinux->写入
  3. 重启计算机，设置BIOS，选择从U盘启动
  4. 出现画面后选择第二项Install Ubuntu
  5. 出现安装引导程序，选择中文，点继续
  6. 选择安装Ubuntu时下载更新
  7. 键盘布局选择English(US)（不选日文键盘就行，结构不大一样）
  8. 设置用户名和密码

　　静待片刻，Ubuntu就装好了。

安装OpenCV

步骤

  1. 官网下载OpenCV源码，选择合适的版本（此处以2.4.13为例）

  2. 解压到任意目录

    unzip opencv-2.4.13.zip

  3. 进入源码路径，创建release目录

    cd opencv-2.4.13
mkdir release

  4. 可以看到在OpenCV目录下有个CMakeLists.txt文件，在编译之前需要安装一些package

    sudo apt-get install build-essential cmake libgtk2.0-dev pkg-config python-dev
 python-numpy libavcodec-dev libavformat-dev libswscale-dev

  5. 进入release目录，安装OpenCV时所有的文件都会被放到这个目录下

    cd release

  6. Cmake编译OpenCV源码，所有的lib文件都会被安装到/usr/local目录下

    cmake -D CMAKE_BUILD_TYPE=RELEASE -D CMAKE_INSTALL_PREFIX=/usr/local ..

  7. 安装

    sudo make install

  8. 测试，在某个目录下建立一个test.cpp文件

    #include <cv.h> 
#include <highgui.h>

using namespace cv;

int main(int argc, char* argv[]) 
{
    Mat image;
    image = imread(argv[1], 1);
    if (argc != 2 || !image.data) {
    printf("No image data\n");
    return -1; }
    namedWindow("Display Image", CV_WINDOW_AUTOSIZE); imshow("Display Image", image);
    waitKey(0);
    return 0;
}

  9. 写一个cmake的makefile，命名为CMakeLists.txt

    project(test)
find_package(OpenCV REQUIRED) 
add_executable(test test) 
target_link_libraries(test ${OpenCV_LIBS}) 
cmake_minimum_required(VERSION 2.8)

  10. 编译并运行，得到可执行文件

    cmake .
make

  11. 随便找一张jpg格式的图片做测试，将该图片（例如为gakki.jpg）放在上面得到可执行文件的路径下，看到输出图片则表示OpenCV安装成功

    test gakki.jpg

可能出现的问题

　　报错：/usr/include/c++/6/cstdlib:75:25:fatal error: stdlib.h: no such file or directory

　　解决办法：安装Ubuntu16.04及以上版本的系统可能遇到此问题，其原因是OpenCV对gcc6的支持不完善，可降低系统自带的gcc和g++版本来解决：

  1. 安装gcc-4.9和g++-4.9

    sudo apt-get install gcc-4.9
sudo apt-get install g++-4.9

  2. 删除自带的gcc和g++

    sudo rm /usr/bin/gcc
sudo rm /usr/bin/g++

  3. 建立软连接

    cd /usr/bin
sudo ln -s gcc-4.9 gcc
sudo ln -s g++-4.9 g++

安装CUDA-8.0

  1. 安装对应kernel版本的kernel header和package development

    sudo apt-get install linux-header-$(uname -r)

  2. 选择runfile安装方式，在CUDA官网下载安装包，根据系统信息选择对应版本的runfile文件

  3. 禁用nouveau，终端输入lsmod | grep nouveau，如果有输出则代表nouveau正在加载

    1. 直接移除这个驱动（备份出来）

      mv /lib/modules/4.4.0-31-generic/kernel/drivers/gpu/drm/nouveau/nouveau.ko /lib/modules/4.4.0-31-generic/kernel/drivers/gpu/drm/nouveau/nouveau.ko.org

    2. 重新加载

      update-initramfs -u

    3. 重启

　　设置完成后可以再次运行lsmod | grep检查禁用是否成功，如果运行和没有任何输出，则代表禁用成功

  1. 重启计算机，到登录界面时，按住Alt+Ctrl+F1，进入text mode，登录账户

  2. 输入sudo service lightdm stop关闭图形化界面

  3. 切换到CUDA安装文件的路径，运行

    sudo sh cuda_8.0.61_375.26_linux.run

  4. 按照提示一步一步操作，首先按Enter键阅读协议并输入accept，然后遇到选择都输入yes，安装成功后会显示installed，否则会显示failed

  5. 输入sudo service lightdm start重新启动图形化界面

  6. 重启电脑，设置环境变量，在终端输入sudo gedit /etc/profile，在打开的文件末尾，添加以下两行

    export PATH=/usr/local/cuda/bin:$PATH
export LD_LIBRARY_PATH=/usr/local/cuda/lib64:$LD_LIBRARY_PATH

  7. 终端输入nvidia-smi，输出显卡信息则说明安装成功

安装cuDNN

  1. NVIDIA官网下载cuDNN，选择cuDNN v5.1 Library fo Linux

  2. 解压下载的压缩包

  3. 把相应的文件拷贝到对应的CUDA目录下

    sudo cp cuda/include/cudnn.h /usr/local/cuda/include/
sudo cp cuda/lib64/libcudnn* /usr/local/cuda/lib64/
sudo chmod a+r /usr/local/cuda/include/cudnn.h
sudo chmod a+r /usr/local/cuda/lib64/libcudnn*

安装MATLAB

  1. 交大的同学可以在交大葡萄上下载MATLAB R2016b，下载后文件夹中包含三个文件：Matlab 2016b Linux64 Crack.rar，R2016b_glnxa64_dvd1.iso，R2016b_glnxa64_dvd2.iso

  2. 在根目录下创建名为matlabLinux的文件夹，并创建两个子文件夹Linux和matlab，其中Linux文件夹用于储存下载的三个文件，而matlab文件夹用于挂载，使用下列命令先挂载R2016b_glnxa64_dvd1.iso

    sudo mount -t auto -0 loop R2016b_glnxa64_dvd1.iso matlab/

  3. 挂载iso之后，会发现系统多了一个盘，说明挂载成功，然后进行安装

    cd matlab
sudo ./install

  4. 选择使用文件安装密钥

  5. 安装密钥填09806-07443-53955-64350-21751-41297

  6. 安装进度进行到80%左右的时候，会弹出一个提示框，请插入dvd2，这时候我们需要重新开一个终端，把dvd2挂载到matlab文件夹中

    sudo mount -t auto -o loop R2016b_glnxa64_dvd2.iso matlab/

  7. 安装完成后取消iso挂载

    umount matlab/

  8. 激活MATLAB

    1. 打开MATLAB

      cd /usr/local/MATLAB/R2016b/bin
./matlab

    2. 载入激活文件license_standalone.lic（位于下载的三个文件中破解文件的解压缩文件夹里）

      可能遇到权限不足的问题，解决方法如下

      cd Crack/     #进入破解文件夹目录
sudo chmod 777 license_standalone.lic     #提高对license_standalone.lic文件的操作权限
cd /usr/local/MATLAB     #进入安装目录
sudo chomd -R 777 R2016b/     #提高对安装目录R2016的操作权限

    3. 把Crack文件夹中R2016b/Linux/R2016b/bin/glnxa64等四个文件复制到/usr/local/MATLAB/R2016b/bin/glnxa64 目录下

      sudo cp Crack/R2016b/bin/glnxa64/lib* /usr/local/MATLAB/R2016b/bin/glnxa64

  9. 打开根目录下的.bashrc文件，sudo gedit ~/.bashrc，在末尾添加

    export PATH="/usr/local/MATLAB/R2016b/bin:$PATH"

    终端输入source ~/.bashrc使命令生效，即可通过在终端输入matlab来启动

安装Caffe

步骤

  1. 在 https://github.com/google/glog 下载glog并安装

    cd glog-master
./autogen.sh
sudo ./configure
sudo make -j8
sudo make install

  2. 安装BLAS和CBLAS

    1. 在官网下载BLAS和CBLAS

    2. 编译BLAS

      cd BLAS-3.7.0
gfortran -c -O3 *.f    # 编译所有的.f文件，生成.o文件
ar rv libblas.a *.o    # 链接所有的.o文件，生成.a文件

    3. 进入CBLAS目录，删除原有的Makefile.in，将Makefile.LINUX重命名为Makefile.in，然后编译

      cd CBLAS
cp ../blas-3.7.0/libblas.a testing    #将上一步编译成功的 libblas.a 复制到 CBLAS目录下的testing子目录
make    #编译所有目录
sudo cp include/cblas.h /usr/include/
sudo cp lib/cblas_LINUX.a /usr/local/libcblas.a
sudo cp testing/libblas.a /usr/local/lib
gcc -shared lib/cblas_LINUX.a testing/libblas.a -o libcblas.so    #将静态库转为动态库
sudo cp libcblas.so /usr/local/
sudo mv /usr/local/libcblas.* /usr/local/lib/

  3. 安装其他依赖项

    sudo apt-get install -y libprotobuf-dev libleveldb-dev libsnappy-dev libopencv-dev libboost-all-dev libhdf5-serial-dev
sudo apt-get install -y libgflags-dev libgoogle-glog-dev liblmdb-dev protobuf-compiler protobuf-c-compiler python-pandas

  4. 在Caffe官网下载caffe

  5. 解压出caffe-master，并进入该文件夹

  6. 拷贝一份Makefile.config.example并重命名为Makefile.config

  7. 修改Makefile.config

    1. 取消注释USE_CUDNN := 1
    2. 取消注释MATLAB_DIR := /usr/local/并修改为MATLAB_DIR := /usr/local/MATLAB/R2016b
    3. 取消注释WITH_PYTHON_LAYER := 1

  8. 编译caffe，终端输入

    cd caffe-master/
sudo make all -j8
sudo make test -j8
sudo make runtest -j8
sudo make pycaffe    #编译Python接口
sudo make matcaffe    #编译MATLAB接口

  9. 把caffe的Python模块接入PYTHONPATH

    sudo gedit ~/.bashrc

    在文件末尾加入export PYTHONPATH=/path/to/caffe/python:$PYTHONPATH，并使配置生效source ~/.bashrc，并在用python命令调出的调试台中输入import caffe，若无报错则成功

  10. 使用MNIST数据集进行测试

    1. 数据预处理

      sh data/mnist/get_mnist.sh

    2. 重建Imdb文件，生成mnist-train-Imdb和mnist-test-Imdb文件夹，这里包含了Imdb格式的数据集

      sh examples/mnist/create_mnist.sh

    3. 训练MNIST

      sh examples/mnist/train_lenet.sh

若无报错则caffe安装成功

可能出现的问题

  1. 安装glog执行./autogen.sh时报错：./autogen.sh: 5: ./autogen.sh: autoreconf: not found

    解决办法：终端输入

    sudo apt-get install autoconf
sudo apt-get install automake
sudo apt-get install libtool

  2. 在sudo make all -j8时报错：

    LD -o .build_release/lib/libcaffe.so.1.0.0
/usr/bin/ld: cannot find -latlas
collect2: error ld returned 1 exit status
make: *** [.build_release/lib/libcaffe.so.1.0.0] error 1

    解决办法：终端输入sudo apt-get install libatlas-base-dev

  3. 在sudo make all -j8时报错：fatal error: hdf5.h: no such file or directory

    解决办法：在makefile_config文件的第94行添加/usr/include/hdf5/serial/到 INCLUDE_DIRS，即将

    INCLUDE_DIRS := $(PYTHON_INCLUDE) /usr/local/include

    改为

    INCLUDE_DIRS := $(PYTHON_INCLUDE) /usr/local/include /usr/include/hdf5/serial/

  4. 在sudo make all -j8时报错：error: cannot find lhdf3_hl and lhdf5

    解决办法：终端输入sudo ldconfig /usr/local/cuda/lib64

  5. 在make runtest时报错：error while loading shared libraries: libcudart.so.8.0: cannot open shared object file: No such file or directory

    解决办法：参考这篇博客和这篇博客

  6. 在执行import caffe时出现错误：ImportError: No module named skimage.io

    解决办法：终端输入

    sudo apt-get install python-pip python-dev build-essential
sudo pip install -U scikit-image

安装Torch

  1. 安装luarocks

    sudo apt-get install luarocks

  2. 安装Torch

    git clone https://github.com/torch/distro.git ~/torch --recursive
cd torch
bash install-deps
curl -s https://raw.githubusercontent.com/torch/ezinstall/master/install-deps | bash
./install.sh

    出现是否自动添加torch路径到.bashrc时，输入yes

  3. 使torch的环境变量生效

    source ~/.bashrc

  4. 终端输入th弹出Torch则安装成功

总结

　　至此我们已经成功安装了深度学习的两个非常常用的框架Caffe和Torch，我本人在安装过程中走了不少弯路，因此写下这篇博客来帮助各位初入坑者。

　　安装过程中可能会有各种各样的bug和error，鉴于大家使用的机器和系统以及环境可能不尽相同，此文仅供参考，遇到问题一定要勤Baidu&Google，祝大家学习科研顺利！
