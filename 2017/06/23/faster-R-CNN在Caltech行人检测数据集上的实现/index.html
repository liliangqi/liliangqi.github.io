faster R-CNN在Caltech行人数据集上的实现

2017-06-23 machine learning

　　这篇文章致力于在faster R-CNN框架下，利用Caffe在Caltech行人数据集上实现行人检测任务。本文分为两部分，首先是对数据集的处理，其次是搭建环境对py-faster-rcnn代码进行复现。

版权声明：本文为博主原创文章，未经博主允许不得转载。

数据集的处理

　　Caltech数据集来自加州理工大学，数据来源于车载摄像头拍摄的街道实景视频。视频总时长约10个小时，分辨率为640×480，30帧/秒，标注了约250,000帧（137分钟），350,000个bounding box和大约2,300个互不相同的行人。该数据集分为11个子集set00~set10，其中set00~set05作为训练集，每个子集大概为1G，均包含6~13个时长为一分钟的seq视频文件；而set06~set11则作为测试集，与训练集属性类似。数据集的注释标签都以vbb文件给出，包括训练集和测试集。

　　要将数据集用于faster R-CNN框架，就要将数据做成类似于PASCAL VOC 2007的格式，也就是图片样本集的名称要是六位数字+.jpg，标签信息以xml文件格式给出，下面我们介绍处理方法。

Seq文件的处理

　　我的工作是基于图片做行人检测，需要将seq文件处理转换成jpg文件。Caltech官网上有 Piotr’s Matlab Toolbox工具用于完成这种转换，其中的功能非常多样，但也非常复杂。如果有同学和我一样，只需要完成简单的转换工作，不妨参考我的代码。

　　首先下载上面提到的官方工具包，解压后得到的目录下有一个叫做videos的文件夹，我们在该路径下添加一个名为processSeq.m的脚本，代码如下：

可以看到，set06就是测试集的一个子集，而V000~V018就是19个时长为一分钟的视频转换成的jpg图片，其中图片名称均为为1.jpg等等。由于我们要用整个数据集来训练和测试，也就是将所有的jpg图片整合到一个文件夹中，所以不能有重名的图片，所以我们第一步工作是进行图片的重命名，Python代码如下：

#coding=UTF-8

import os
import re 

re_digits = re.compile(r'(\d+)')  

def emb_numbers(s):  
    pieces=re_digits.split(s)  
    pieces[1::2]=map(int,pieces[1::2])      
    return pieces   

def sort_strings_with_emb_numbers(alist):
    '''根据字符串内嵌的数字进行排序，例如f2应在f10之前'''
    return sorted(alist, key=emb_numbers)


class BatchRename():
    '''批量重命名文件夹中的图片文件'''

    def __init__(self):
        '''设置目标文件夹的绝对路径'''
        self.path = '/Users/liliangqi/Desktop/new'

    def rename(self):
        '''根据文件后缀修改特定的文件名，可以设定命名后的文件名长度'''
        filelist = os.listdir(self.path)
        if '.DS_Store' in filelist:
            filelist.remove('.DS_Store') # macOS系统的文件夹中往往会出现这个文件
        filelist = sort_strings_with_emb_numbers(filelist)

        name_length = 0 # 文件名的长度，不含后缀，如'001.jpg'文件名长度为3
        total_num = len(filelist)
        i = 1 # 命名起始编号

        for item in filelist:
            if item.endswith('.jpg'): # 此处可改为'.png'，那么png图片将被转化为jpg
                if name_length:
                    new_name = '0'*(name_length-len(str(i))) + str(i) + '.jpg'
                else: # 可将name_length设为0，从而避免统一的文件名长度
                    new_name = str(i) + '.jpg'

                src = os.path.join(os.path.abspath(self.path), item)
                dst = os.path.join(os.path.abspath(self.path), new_name)
                try:
                    os.rename(src, dst)
                    print('converting %s to %s ...' % (src, dst))
                    i = i + 1
                except:
                    continue

        print('total %d to rename & converted %d jpgs' % (total_num, i - 1))



if __name__ == '__main__':
    demo = BatchRename()
    demo.rename()

该代码每次处理一个子集，所以在处理下一个子集的时候，要看刚刚处理完的图片命名编号是多少，比如是001128.jpg，那么接下来就从001129开始命名，也就是将第33行改为i = 1129即可。

　　处理了整个数据集后得到249,884张jpg图片，命名为000001.jpg到249884.jpg，接下来处理标签文件。

Vbb文件的处理

　　Caltech数据集的标签信息以vbb文件的形式给出，亦即video bounding box文件。首先在官网上下载Matlab evaluation/labeling code (3.2.1)，这个包是用于处理vbb文件的，但是官方给的包都挺坑的，给了一堆乱七八糟的函数，偏偏没有转换的功能，所以我们需要自己在这个包中再添加一个脚本文件：（代码来源于dachao）

% this function is to convert vbb file to txt  
%by dachao at 2015.11.27  
a = [0 1];  
vPath = '/Users/liliangqi/Downloads/annotations/';  
outpath = '/Users/liliangqi/Downloads/annotations/converted/';   
if ~isdir(outpath)  
    mkdir(outpath);  
end  
disp('Begin to extract annotations from vbb....')  
for ii = 1:length(a)  
    vName = ['set0',num2str(a(ii))];    
    vbbpath = [vPath vName];  
    str = dir(vbbpath);  
    for k = 3:numel(str)  
        finalpath = [vbbpath,'/',str(k).name];   
        disp(finalpath);  
        A = vbb( 'vbbLoad', finalpath);    
        Path = [outpath,'/',vName];  
        if ~isdir(Path)  
            mkdir(Path);  
        end  
        st = str(k).name;  
        p = findstr(st,'.');  
        st = st(1,1:p(1)-1);  
        fnm = [st,'.txt'];       
        c = fopen([Path,'/',fnm],'w');    
        for i = 1:A.nFrame    
            iframe = A.objLists(1,i);    
            iframe_data = iframe{1,1};    
            n1length = length(iframe_data);    
            for  j = 1:n1length    
                iframe_dataj = iframe_data(j);    
                if iframe_dataj.pos(1) ~= 0  %pos  posv    
                    fprintf(c,'%d %f %f %f %f\n', i, iframe_dataj.pos(1),...  
                     iframe_dataj.pos(2),iframe_dataj.pos(3),iframe_dataj.pos(4));    
                end    
            end    
        end    
    end  
fclose(c);   
end  
disp('Done................')

这个脚本中需要注意的是第3行，a = [0 1]表示你所选择的子集，结合第11行，如果a = 0就表示选择要转换的子集是set00，如果要转换子集set10，则将第3行改为a = 0，第11行改成vName = ['set1',num2str(a(ii))];即可。转换后得到txt文件如下图所示：

第一列是图片编号，第二列到第五列是行人bounding box的信息，依次是x坐标、y坐标、box的宽度和高度。每个vbb文件对应一个seq文件，所以我们需要像重命名图片文件一样，对txt文件的第一列图片编号进行重命名，然后将所有的txt文件整合成一个大的txt文件。这里我的处理方式略显笨拙，如果哪位同学有更好的方法，请与我联系讨论，我会将你的好办法贴出来供更多的人学习。

  1. 首先将txt文件导入Excel，得到csv文件或者xsl文件
  2. 在MATLAB中导入xls或csv形式的表格成为矩阵，矩阵共五列，与上述txt文件相似
  3. 利用MATLAB的for循环代码对矩阵第一列进行加数值操作，比如set00的V000.txt文件的最后一行图片编号是1845，那么在处理V001.txt文件转换成的矩阵时，就给其第一列的每一个数字都加1即可
  4. 将处理完的矩阵利用MATLAB的内置函数csvwrite()保存为csv文件，方便用Excel进行处理
  5. 把所有的csv文件拼接起来，形成一个大表格，并将第2列加到第4列上，将第3列加到第5列上，这时bounding box的信息就变成了(x_min, y_min, x_max, y_max)
  6. 将第2列到第5列向后平移一列，空出第二列全部输入文本person
  7. 将第一列所有内容由原本的六位数字，改成六位数字+.jpg，如000001改成000001.jpg，这个功能很容易实现，大家百度一下，我就不细说了
  8. 把保存这个大表格保存成制表符间隔的txt文本文件
  9. 打开txt文件，利用查找&替换将所有制表符替换成空格键，得到如下txt文件：

训练集和测试集的设置

　　标准数据集已经生成了，接下来就是指定训练集和测试集了。虽然官方说set00~set05作为训练集，其余作为测试集，但它的前提是那个时候只有前六个子集有标签信息，而现在我们拥有所有子集的标签信息，就无所谓那些做训练集那些做测试集了，那么最好的设置方法，就是随机设定。

　　PASCAL VOC 2007标准要求的指定训练集和测试集的方式是用txt文件列出作为训练（测试集）的图片编号，我们利用如下的代码来实现随机设定训练集和测试集：

%trainval占总数据集的50%，test占总数据集的50%；train占trainval的50%，val占trainval的50%；
%上面所占百分比可根据自己的数据集修改，如果数据集比较少，test和val可少一些


%%

    end
    if ismember(i, test)
        fprintf(ftest,'%s\n',xmlfile(i+2).name(1:end-4));
    end
end
fclose(ftrainval);
fclose(ftrain);
fclose(fval);
fclose(ftest);

利用faster R-CNN进行行人检测

配置实验环境

步骤

  1. 将faster R-CNN工程从github上克隆到本地

    git clone --recursive https://github.com/rbgirshick/py-faster-rcnn.git

  2. 安装必要的Python包： cython, python-opencv, easydict

    sudo apt-get install cython
sudo apt-get install python-opencv
sudo apt-get install python-pip python-dev build-essential    #安装pip
sudo pip install easydict
sudo pip install numpy

  3. 我们将克隆faster R-CNN工程的根目录称为FRCN_ROOT，然后build Cython模型

    cd $FRCN_ROOT/lib
make

  4. 首先对caffe-fast-rcnn的Makefile.config文件进行配置，可以参考我的Ubuntu 16.04安装caffe及torch一文，然后build caffe 以及pycaffe

    cd $FRCN_ROOT/caffe-fast-rcnn
sudo make all -j8
sudo make pycaffe

  5. 下载预训练的检测模型

    cd $FRCN_ROOT
./data/scripts/fetch_faster_rcnn_models.sh
./data/scripts/fetch_imagenet_models.sh

    经此步骤我们在 $FRCN_ROOT/data 目录下将会得到一个叫做 faster_rcnn_models的文件夹和一个叫做imagenet_models的文件夹。

  6. 运行Demo

    cd $FRCN_ROOT
./tools/demo.py

    此Demo使用的网络为VGG16网络，利用训练好的模型在五幅图片上做测试，输出带有bounding box的结果。

可能出现的问题

  1. 在sudo make all -j8时报错：

    In file included from ./include/caffe/util/cudnn.hpp:5:0,
                 from ./include/caffe/util/device_alternate.hpp:40,
                 from ./include/caffe/common.hpp:19,
                 from src/caffe/data_reader.cpp:6:
/usr/local/cuda/include/cudnn.h:803:27: note: declared here
 cudnnStatus_t CUDNNWINAPI cudnnSetPooling2dDescriptor(

    原因：我们使用的cudnn v5中修改了cudnnStatus_t CUDNNWINAPI cudnnActivationBackward等函数接口，而旧版的caffe中（例如faster rcnn里的caffe）仍然使用原来的调用方式。

    解决办法：使用最新版本的caffe里面的部分文件对faster rcnn文件进行替换

    cp caffe-master/inlude/caffe/layers/cudnn_* caffe-fast-rcnn/include/caffe/layers/
cp caffe-master/src/caffe/layers/cudnn_* caffe-fast-rcnn/src/caffe/layers/
cp caffe-master/include/caffe/util/cudnn.hpp caffe-fast-rcnn/include/caffe/util/

训练PASCAL VOC 2007数据集

步骤

  1. 下载数据集

    wget http://host.robots.ox.ac.uk/pascal/VOC/voc2007/VOCtrainval_06-Nov-2007.tar
wget http://host.robots.ox.ac.uk/pascal/VOC/voc2007/VOCtest_06-Nov-2007.tar
wget http://host.robots.ox.ac.uk/pascal/VOC/voc2007/VOCdevkit_08-Jun-2007.tar

  2. 将下载的压缩文件解压缩到一个名为VOCdevkit的文件夹中

    tar xvf VOCtrainval_06-Nov-2007.tar
tar xvf VOCtest_06-Nov-2007.tar
tar xvf VOCdevkit_08-Jun-2007.tar

  3. 建立软连接

    cd $FRCN_ROOT/data
ln -s $VOCdevkit VOCdevkit2007

    其中$VOCdevkit表示我们刚刚解压出的VOCdevkit文件夹的路径，这一步操作其实是方便我们共享数据，比如我在两个工程里都要用到VOCdevkit文件夹里的数据，我就不必把VOCdevkit文件夹拷贝两份放到每个工程里，而是通过这种数据共享的方式来节省硬盘内存。当然了这样做的缺点就是你在每个共享软连接的目录下修改文件时，所有同时共享的文件夹和原始文件夹中的数据都会被同步修改。

  4. 选择训练方式以及训练模型

    faster R-CNN提供了两种训练方式：分阶段训练和端到端训练。分阶段训练就是先训练用于生成region preposal的rpn网络，再训练用于检测的fast R-CNN网络；而端到端的网络则是连续进行训练，无需人为地接续后面的操作。所以我们这里选择端到端的训练方式。

    在终端运行：

    ./experiments/scripts/faster_rcnn_end2end.sh 0 VGG16 pascal_voc

    其中0表示使用的GPU编号为0，VGG16表示使用VGG16网络，pascal_voc表示使用的数据格式为PASCAL VOC格式的，恰好我们这里使用的数据集就是PASCAL VOC 2007数据集。

  5. 训练结束后我们会在$FRCN_ROOT目录下得到一个output文件夹，里面保存着我们训练后得到的模型

可能出现的问题

  1. 训练模型时出现错误：TypeError: slice indices must be integers or None or have an __index__ method

    原因：numpy版本过高，将其降级到1.11.0即可

    解决办法：

    sudo pip install -U numpy==1.11.0
